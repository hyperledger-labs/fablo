Title: Port collision when multiple orgs have many orderer instances
File: src/extend-config/extendOrgsConfig.ts
Line: 264-271
Impact: HIGH - Docker containers fail to start due to port binding conflicts when the network has more than a few organizations.
Root Cause: The function getPortsForOrg allocates ports using a fixed stride of 20 per org index (e.g. caPort = 7020 + 20*orgIndex). With multiple orderers and peers, the port ranges can overlap between adjacent orgs. For example, if org 0 has 10 orderer instances (ports 7030-7039), org 1's CA port starts at 7040, but org 0's peer ports start at 7021 and could reach 7030+ with enough peer instances, colliding with orderer ports.
Suggested Fix: Calculate total port consumption per org dynamically and assign non-overlapping ranges, or validate for port collisions during the extend phase and emit an error.
